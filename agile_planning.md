# Agil Projektplanering: Bönan & Koppen

Detta dokument innehåller planeringen för utvecklingen av webbappen "Bönan & Koppen" enligt en agil process.

## 1. Mål och Vision

*   **Vision:** Att bli den ledande online-plattformen för kaffeentusiaster som värdesätter kvalitet, hållbarhet och spårbarhet i sitt kaffe.

*   **Syfte & Mål:**
    *   Erbjuda ett kuraterat urval av högkvalitativa kaffebönor från lokala rosterier.
    *   Göra det enkelt för kunder att upptäcka och köpa unika kaffesorter.
    *   Skapa en transparens kring kaffets ursprung och rostningsprocess.

*   **Målgrupp:** Medvetna konsumenter (25-50 år) med ett genuint intresse för kaffe. De är villiga att betala mer för en premiumprodukt och värdesätter hållbarhet och att stödja lokala företag.

*   **Problem som löses:** Det är svårt och tidskrävande för kaffeälskare att hitta och köpa unika, högkvalitativa och hållbart producerade kaffesorter från olika lokala rosterier. "Bönan & Koppen" samlar de bästa på ett ställe.

## 2. User Stories

*   **Epics (övergripande teman):** Användarhantering, Produkthantering, Köpprocess

1.  **(Användarhantering)** Som *ny besökare* vill jag kunna *skapa ett konto* för att kunna spara mina uppgifter och se min orderhistorik.
2.  **(Användarhantering)** Som *registrerad användare* vill jag kunna *logga in* på mitt konto för att få en personlig upplevelse.
3.  **(Produkthantering)** Som *kund* vill jag kunna *se alla tillgängliga kaffesorter* på en produktsida.
4.  **(Produkthantering)** Som *kund* vill jag kunna *klicka på en produkt* för att se detaljerad information, som ursprung, smakprofil och rostningsdatum.
5.  **(Köpprocess)** Som *kund* vill jag kunna *lägga en produkt i min varukorg*.
6.  **(Köpprocess)** Som *kund* vill jag kunna *se och redigera min varukorg* innan jag betalar.
7.  **(Köpprocess)** Som *kund* vill jag kunna *gå till kassan* och fylla i mina leveransuppgifter.
8.  **(Köpprocess)** Som *kund* vill jag kunna *betala för min beställning* med ett kreditkort.
9.  **(Användarhantering)** Som *inloggad kund* vill jag kunna *se min tidigare orderhistorik* på min profilsida.

## 3. Product Backlog

Enkel, prioriterad backlog baserad på user stories. Prioriteringen fokuserar på att snabbt möjliggöra ett komplett köpflöde för en kund (Minimum Viable Product).

| Prioritet | User Story | Epic | Kommentar |

| **1 (Högst)** | 3. Se alla tillgängliga kaffesorter | Produkthantering | Kärnfunktionalitet. Utan detta ser kunden inget. |
| **2** | 4. Se detaljerad produktinformation | Produkthantering | Viktigt för att kunden ska kunna ta ett köpbeslut. |
| **3** | 5. Lägga en produkt i varukorgen | Köpprocess | Nödvändig för att kunna handla. |
| **4** | 6. Se och redigera varukorgen | Köpprocess | Standardfunktion som skapar trygghet. |
| **5** | 7. Gå till kassan & ange leverans | Köpprocess | Kritiskt för att kunna slutföra köpet. |
| **6** | 8. Betala med kreditkort | Köpprocess | Kritiskt för att kunna slutföra köpet. |
| **7 (Hög)** | 1. Skapa ett konto | Användarhantering | Viktigt, men man kan tänka sig köp utan konto initialt. |
